Introduction


Hi there! I'm the creator of this website. I go online as JBMagination, but most call me JBM.

You've probably seen the many Hour of Code websites out there. Unfortunately, most of the "code" that it writes is all fake.

That HTML, CSS, and JavaScript you write with? It's probably not real. Using blocks? The "JavaScript" it gives back is also not real.

And sure, that could make things much easier to write, but I always wanted to write real code when using those code websites.

And that's why I created this. To give newcomers an easy way to learn basic code.

If you truly want to get deep into learning how to code, here's some advice.


Don't be afraid to look stuff up

While you should never use copy and paste, you should definitely look stuff up if you have to. Most people who code do it all the time!

You should always try to do it yourself first before searching, however. If you feel truly stuck, you should definitely search.


Take breaks if you need to

It's never healthy to look at the screen for too long. Make sure that you take a break every once in a while, at least every few hours.

While it can be fun to code, always prioritize your health first.


Coding is not the new literacy

Coding is not an essential part of your life. Many make great heaps of money from coding, but you shouldn't feel forced to learn it.

There is a difference between literacy and code literacy.


Know the differences between computer science and computer programming

While computer programming and computer science are closely tied together, there is a big difference between them.

Computer programming is a subset of computer science, and is largely focused on, well, creating and utilizing code to make programs.

Computer science, on the other hand, is largely focused on studying data and data interaction, alongside the design of computers.

Consider Peter Norvig as a computer scientist. He studies in the field of artificial intelligence and is the directory of research at Google.

Consider Jeff Atwood as a computer programmer. He created the developer platform Stack Exchange, but he didn't really have to study many fields to be considered a programmer.


There are plenty of sources out there

If this sort of work doesn't fit your boat, I recommend W3Schools.

They offer different courses than I do, but most of what you'll want to know is also offered both here and there, so go give them a visit if this doesn't seem to work out for you.
